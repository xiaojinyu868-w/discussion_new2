## 💡 我们解决了什么问题？

我们认为下一代组织协作范式应该是 **AI 原生**的。会议是组织协作中信息密度最高的场景，因此我们选择从会议切入。

传统会议存在七大痛点：**记不住、听不懂、对不齐、留不下、思维僵局、决策难题、聚焦涣散**。根本原因是——会议至今仍是"纯人力驱动"，没有充分引入 AI 能力。

**MeetMind** 是一个实时上下文驱动的 AI 原生会议协作者。它不仅做毫秒级语音转写，更能：
- **实时捕捉潜台词**，揭示"表面共识、实际分歧"
- **主动推送洞察**，在跑题、冗余、决策卡点时及时介入
- **视觉化共识生成**，利用 Nano Banana Pro 将抽象讨论实时转化为图表、海报、创意图

我们让会议从**"过耳即忘"的音频流**，变成**"一眼即懂"的视觉资产**——颠覆组织信息对齐效率。

## 👥 用户是谁？

高频开会的**创新创业团队、产品经理、远程协作组织**——他们对 AI 应用有迫切需求，却苦于现有工具无法真正融入协作流程。

**想要达到什么目的？**

将 AI 能力**无缝嵌入组织协作**，实现三个核心目标：
1. **消除"表面共识"**——让隐性分歧在会议中就被发现和解决
2. **沉淀"组织记忆"**——每场会议自动形成可检索、可追溯的知识资产
3. **脑暴即产出**——让每一次头脑风暴都能直接生成可传播的视觉结果

## 🧩 已实现功能：三模态人机交互系统

### 模式一：人主动触发（6 个技能按钮）

用户点击按钮，AI 立即响应：

| 类别 | 技能 | 功能说明 |
|------|------|----------|
| 🔮 AI 技能 | **潜台词** | 分析发言者的潜在意图，揭示"话外之音" |
| ⚡ AI 技能 | **灵感** | 基于当前讨论内容，激发创意和新思路 |
| 🎯 AI 技能 | **聚焦** | 检测跑题并提醒，总结当前核心议题 |
| 📊 视觉化 | **数据图表** | 将口头数据转为雷达图/柱状图/流程图 |
| 🎨 视觉化 | **创意图像** | 生成概念图/示意图/创意配图 |
| 📋 视觉化 | **逻辑海报** | 生成结构化信息海报，一图胜千言 |

### 模式二：AI 主动推流（Agent-Flow 智能联动）

Agent 每 **10 秒**自动巡检上下文，检测到以下情况时**主动推送洞察**：

| 检测类型 | 冷却时间 | 触发条件 | 自动动作 |
|----------|----------|----------|----------|
| 📈 数据洞察 | 60s | 检测到数字/百分比/对比/趋势词 | 自动生成数据图表 |
| ✅ 决策建议 | 90s | 检测到"确定/定了/同意"等决策词 | 生成决策记录卡片 |
| 🎯 跑题提醒 | 120s | 语义偏离主议题 | 自动触发聚焦提醒 |
| 🔄 冗余警告 | 180s | 重复讨论相同内容 | 生成精简建议 |
| 📝 周期性总结 | 30s | 定时触发 | 生成阶段小结 |

**核心机制**：
- **内容哈希去重**：基于 MD5 的幂等机制，防止重复洞察
- **独立冷却时间**：不同检测类型独立冷却，避免信息过载
- **转写-洞察关联**：双向关联，点击可跳转追溯

### 模式三：上下文自由问答（记忆盒子）

基于 Context Stream 的 RAG 检索，用户可**随时提问**：

- "刚才谁提到了预算？"
- "前 10 分钟的核心结论是什么？"
- "把刚才的方案对比做成表格"
- "总结一下小明的所有发言"

**技术实现**：
- 全量转写文本实时存入"记忆盒子"
- 后台 Agent 持续向量化处理
- 问答时优先从长期记忆库检索相关上下文

### 核心技术点

| 技术 | 实现细节 |
|------|----------|
| **实时联动闭环** | 10 秒级：音频切片 → 转写 → Agent 分析 → 动作分发 → 前端渲染 |
| **内容哈希去重** | 基于 MD5 的内容哈希，相同内容不重复触发洞察 |
| **独立冷却机制** | 数据(60s)/决策(90s)/跑题(120s)/冗余(180s) 独立冷却 |
| **Context Stream** | 全量上下文存储 + 向量化，支持任意时刻 RAG 检索 |
| **WebSocket 双向通信** | 单连接完成音频推流 + 转写下行 + 洞察推送 |

## 🚀 快速开始

### 环境要求
- Node.js ≥ 18，npm ≥ 9
- 阿里云通义听悟账号（AccessKeyId/Secret、AppKey）
- Qwen API Key
- Nano Banana Pro API Key（图像生成）

### 后端运行

```bash
cd backend
npm install
```

创建 `.env` 文件：

```env
PORT=4000
TINGWU_REGION=cn-beijing
TINGWU_ACCESS_KEY_ID=你的AK
TINGWU_ACCESS_KEY_SECRET=你的Secret
TINGWU_APP_KEY=你的AppKey
QWEN_API_KEY=你的Qwen密钥
NANO_API_KEY=你的Nano密钥
```

启动服务：

```bash
npm start
```

### 前端运行

```bash
cd frontend
npm install
npm run dev
```
